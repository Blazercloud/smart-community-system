# 🏙️ 东软智慧社区前端开发说明文档

## 📘 项目概述

**项目名称：** 东软智慧社区平台
 **项目类型：** 前后端分离 Web 应用
 **前端框架：** Vue 3.2 + Vite 4 + Element Plus + Pinia + Axios
 **后端接口：** Spring Boot 3 + MyBatis Plus + JWT + MySQL
 **项目目标：** 为社区居民提供集购物、生活缴费、报修、访客登记等一体化智慧服务平台。

------

## 🧱 项目结构

```
smart-community-frontend/
├── README.md
├── package.json
├── vite.config.js
├── src/
│   ├── api/                # 接口请求封装（Axios）

│   ├── components/         # 公共组件
│   ├── router/             # 路由配置（区分用户端/管理端）
│   ├── store/              # Pinia 状态管理
│   ├── utils/              # 工具函数（auth、storage等）
│   ├── views/
│   │   ├── user/           # 用户端模块
│   │   │   ├── mall/       # 商城服务
│   │   │   ├── community/  # 社区服务
│   │   │   ├── account/    # 用户账户中心
│   │   ├── admin/          # 管理端模块
│   │   │   ├── merchant/   # 商户后台
│   │   │   ├── property/   # 社区后台
│   └── App.vue
└── public/
```

## 🧩 模块划分与子模块设计

### 🌐 一、用户端（User Web）

#### 1️⃣ 商城服务模块 `/views/user/mall/`

| 子模块         | 功能描述                                                 |
| -------------- | -------------------------------------------------------- |
| **首页**       | 展示全部商品、促销商品；显示用户登录状态                 |
| **商品搜索**   | 支持商品名、简介模糊查询                                 |
| **商品详情页** | 展示商品图片、价格、库存、门店信息；支持加入购物车与收藏 |
| **购物车**     | 增减数量、移除商品、批量结算                             |
| **订单管理**   | 创建订单、支付订单、查看待付款/已付款/待取货/已完成订单  |
| **商品收藏**   | 收藏与取消收藏功能                                       |

#### 2️⃣ 用户账户中心 `/views/user/account/`

| 子模块        | 功能描述                             |
| ------------- | ------------------------------------ |
| **登录/注册** | 手机号+密码登录注册、验证码找回密码  |
| **个人资料**  | 查看与修改头像、手机号、用户名、邮箱 |
| **修改密码**  | 修改登录密码                         |
| **我的收藏**  | 查看已收藏商品                       |
| **钱包管理**  | 钱包余额查询、充值、转账、账单明细   |

#### 3️⃣ 社区服务模块 `/views/user/community/`

| 子模块       | 功能描述                           |
| ------------ | ---------------------------------- |
| **公告中心** | 浏览公告列表                       |
| **访客登记** | 登记访客姓名、联系方式、来访目的等 |
| **车位管理** | 查看车位信息、绑定车牌号           |
| **报事报修** | 提交维修申请，上传图片、描述       |
| **事项投诉** | 提交投诉内容，查看处理进度         |
| **生活缴费** | 物业费线上缴纳、支付记录查询       |

| 模块         | 子模块         | 功能描述                       |

| 模块         | 子模块       | 功能描述               |
| ------------ | ------------ | ---------------------- |
| **公告管理** | 发布公告     | 向用户端发布社区公告   |
| **投诉管理** | 投诉处理     | 查询并处理用户投诉     |

| **车位管理** | 车位信息统计 | 查询用户车位使用数据   |
| **物业管理** | 报事处理     | 处理报修申请，更新状态 |
| **缴费管理** | 缴费记录     | 查询用户缴费明细       |

## 📡 接口依赖说明

| 模块     | 主要接口路径示例                                         | 请求方式   |
| -------- | -------------------------------------------------------- | ---------- |
| 登录注册 | `/api/auth/login`, `/api/auth/register`                  | POST       |
| 商品管理 | `/api/product/list`, `/api/product/detail/{id}`          | GET        |
| 购物车   | `/api/cart/add`, `/api/cart/delete`, `/api/cart/list`    | POST / GET |
| 订单管理 | `/api/order/create`, `/api/order/pay`, `/api/order/list` | POST / GET |
| 报修投诉 | `/api/community/repair`, `/api/community/complaint`      | POST / GET |
| 公告发布 | `/api/admin/notice`                                      | POST / GET |

------

## 🧑‍💻 开发规范

- 所有接口统一通过 `api/` 模块封装。
- 全局状态管理使用 `Pinia`。
- 接口权限控制通过路由守卫（`router.beforeEach`）实现。
- 所有页面统一使用 `Element Plus` 组件风格。
- 按模块划分路由与组件目录，避免交叉依赖。
